## Issue 2 — Audit ledger core + screen (AFK)

### Parent
PRD-001.

### What to build
An append-only audit service (pure module): each entry records actor/agent, action, timestamp, payload hash, and previous-entry hash (SHA-256), forming a verifiable chain; plus a dedicated Audit Ledger screen listing entries generated by real agent actions in the demo flows, with per-agent attribution and a "Verify integrity" action that walks the chain and reports intact/broken. Session-scoped persistence (in-memory/localStorage); no server.

### Acceptance criteria
- [ ] Unit tests: appending N entries yields a chain that verifies; mutating any historical entry fails verification
- [ ] Agent decisions in the existing demo flows write ledger entries automatically
- [ ] Ledger screen shows entries with agent, action, time, hash lineage; "Verify integrity" reports status
- [ ] Existing flows behave identically apart from logging

### Blocked by
Issue 1.
